# Project Initialization Rules

**Rule Name:** R0 (Always run first)

## Startup Sequence
1. **Update skills** — refresh/sync all skill definitions before starting work.
2. **Use Caveman skill (Full Mode)** — apply token-reduction behavior at maximum strictness for this session.
3. **Environment** — development PC is Windows 11. Assume Windows paths, PowerShell syntax, and Windows-specific tooling (IIS, appcmd.exe, etc.) unless told otherwise.
4. **Skill matching** — scan the project type and load all relevant skills for it (e.g. nopCommerce, MAUI, docx/xlsx/pdf, frontend-design, etc.).
5. **Gap check** — if a needed skill is missing or doesn't exist yet, flag it and suggest creating one (via skill-creator) instead of proceeding without it.

## Notes
- This block runs once at the start of a project/session, before the GIT workflow rules (R1) or any other rule set applies.
