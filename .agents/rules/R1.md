# Git Workflow Rules

**Rule Name:** R1 (Default)

## Branch Flow
dev → feature/{name} → features/{yy-mm-dd} (daily collection) → dev → main

## Branch Naming
- Feature branch: `feature/{feature-name}` (created from `dev`)
- Daily collection branch: `features/{yy-mm-dd}`

## Default Behavior
- Always work on the current feature branch unless told otherwise.
- When a feature is complete:
  1. Run tests.
  2. If tests pass, commit the feature branch.
  3. Merge the feature branch into today's daily collection branch (`features/{yy-mm-dd}`) — create it from `dev` if it doesn't exist.
  4. Push the daily collection branch.
- Do NOT merge or push if tests fail — report the failure instead.

## Voice Commands / Commands
- **"go to dev"** → Merge the current daily collection branch (`features/{yy-mm-dd}`) into `dev`, then push `dev`.
- **"go to main"** → Merge `dev` into `main`, then push `main`.
